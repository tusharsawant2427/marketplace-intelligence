# Domain Model

This document outlines the core domain models that represent the business entities in the system. Rather than treating data as raw JSON, dictionaries, or generic database rows, these are strictly typed domain models that encapsulate the state, logic, and behaviors necessary to answer the business questions.

Each domain model serves as an abstraction layer over the actual underlying database tables, unifying fragmented data into a cohesive structure for the AI.

## Core Models

### 1. Listing
Represents the product listing on a specific marketplace.
- **Listing ID:** Internal unique identifier.
- **ASIN:** Amazon Standard Identification Number.
- **FSIN:** Flipkart Standard Identification Number.
- **Marketplace:** The platform (e.g., Amazon, Flipkart).
- **Status:** Current state (e.g., Active, Inactive, Suppressed).
- **Metadata:** Core listing information like title, description, and attributes.

**Data Source Tables:** `listings`, `online_selling_products`, `listing_marketplaces`, `listing_meta_data`, `platforms`

### 2. Pricing
Represents the financial and pricing strategy for a listing.
- **Selling Price:** Current active price on the marketplace.
- **MRP:** Maximum Retail Price.
- **Cost:** Total landed cost of the product.
- **Minimum Price:** The floor price allowed to maintain the required minimum margin.
- **Recommended Price:** The AI's suggested optimal selling price.

**Data Source Tables:** `products`, `saleable_mrps`, `production_costs`, `online_selling_products`

### 3. Charges
Represents all fees and deductions associated with selling a product.
- **Commission:** Marketplace category referral/platform fees.
- **Shipping:** Fulfillment, handling, and transit fees.
- **Storage:** Warehousing and long-term storage fees.
- **Advertisement:** Attributed PPC and promotional spend.
- **Royalty:** Licensing, franchise, or brand royalties.

**Data Source Tables:** `meesho_shipping_fees`, `jiomart_commission_fees`, `jiomart_shipping_fees`, `jiomart_storage_fees`, `saleable_royalty_analyses`, `monthly_advertisements`

### 4. SalesHistory
Represents historical performance and traffic metrics.
- **Orders:** Total number of orders placed.
- **Revenue:** Total gross sales generated.
- **Units:** Total number of units sold.
- **CTR:** Click-Through Rate.
- **Conversion:** Conversion rate.

**Data Source Tables:** `amazon_orders`, `amazon_order_items`, `flipkart_orders`, `flipkart_order_items`, `ecommerce_sale_orders`, `product_wise_net_sales_reports`

### 5. MarketplaceSnapshot
Represents a point-in-time capture of external data.
- **Data:** Everything fetched from Amazon/Flipkart APIs (e.g., current Buy Box owner, competitor prices, immediate inventory levels, live platform status).

**Data Source:** Live Marketplace APIs (Amazon SP-API, Flipkart API) and cached snapshot tables.

## Agent Communication Models

### 6. AnalysisRequest
Represents the structured input/payload given to an intelligent agent.
- **Question:** The specific business question to be answered (e.g., "Why is this listing making a loss?").
- **Listing Id:** The target listing to analyze.
- **Marketplace:** The platform context.
- **Capability:** The required intelligence domain (e.g., Pricing Intelligence, SEO Intelligence).

### 7. AnalysisResult
Represents the standardized, structured output generated by an intelligent agent.
- **Summary:** A high-level overview of the issue.
- **Root Cause:** The underlying reason identified by the AI.
- **Confidence:** AI's certainty score in its analysis.
- **Recommendations:** Actionable steps to resolve or improve the situation.
- **Priority:** Urgency level indicating how soon the recommendations must be acted upon.
